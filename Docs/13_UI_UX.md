# Objectif

Définir les principes d'ergonomie et de présentation de TarotScore V2.

Ce document garantit une interface homogène sur l'ensemble de l'application.

---

# Principes

L'application doit être :

- simple
- rapide
- lisible
- cohérente
- utilisable sur écran tactile comme à la souris

L'utilisateur ne doit jamais avoir à se demander quelle est l'étape suivante.

---

# Navigation

La navigation doit rester peu profonde.

Objectif :

Accueil

↓

Fonction

↓

Action

Maximum trois niveaux.

---

# Écrans

Chaque écran possède :

- un titre
- une action principale
- un contenu clair
- un bouton Retour si nécessaire

---

# Boutons

Chaque écran possède une action principale clairement identifiable.

Exemples :

Ajouter

Enregistrer

Démarrer

Valider

---

# Formulaires

Les formulaires doivent :

- valider immédiatement les erreurs
- afficher des messages explicites
- limiter les champs obligatoires

---

# Messages

Les messages doivent être :

- courts
- explicites
- positifs

Exemple :

"Le joueur a été enregistré."

Éviter :

"Opération terminée."

---

# Confirmation

Une confirmation est demandée uniquement pour les actions irréversibles.

Exemples :

Suppression

Réinitialisation

Import d'une sauvegarde

---

# Icônes

Les icônes suivent Material Design.

Toujours accompagner une icône importante d'un libellé.

---

# Couleurs

Le thème est défini dans AppTheme.

Les couleurs ne sont jamais codées directement dans les widgets.

---

# Responsive

L'application doit rester utilisable :

- sur téléphone
- sur tablette
- sur ordinateur

Les écrans larges utilisent davantage l'espace disponible.

---

# Accessibilité

Les textes doivent rester lisibles.

Les contrastes doivent être suffisants.

Les zones cliquables doivent être confortables.

---

# Philosophie

La simplicité prime toujours sur la richesse visuelle.

L'utilisateur doit pouvoir enregistrer une donne en quelques secondes sans quitter la table de jeu.
